## How Can I Contribute?

### Reporting Bugs

Found a bug? That's ironic, since bugs are exactly the kind of thing that should trigger our sad trombone feature.

**Before submitting a bug report:**
1. Check if your "bug" is actually just JavaScript being JavaScript
2. Verify you're not running Internet Explorer 6
3. Ensure the universe hasn't fundamentally changed since you last checked

**When submitting a bug report, include:**
- Clear description of the problem
- Steps to reproduce (be specific)
- Expected behavior vs. actual behavior
- Screenshots of violins (if applicable)
- Your emotional state at the time of discovery

### Suggesting Features

Have an idea for a new feature? We'd love to hear it! Unless it's:
- Adding actual sympathy (out of scope)
- Making the violin bigger (defeats the purpose)
- Removing sarcasm (absolutely not)

### Adding Violin Memes

This is the most valuable contribution you can make!

1. Find a high-quality tiny violin meme/GIF
2. Ensure it's appropriately tiny
3. Verify it conveys the right amount of "sympathy"
4. Add the URL to the `violinMemes` array in `MicroViolin.js`
5. Test that it displays correctly
6. Submit a PR with the title: "Added violin #[number]"

- ‚úÖ Bonus points for sarcasm

## Style Guidelines

### JavaScript Style

We follow a strict coding style because we're professionals:

1. **Indentation:** 4 spaces (fight us)
2. **Quotes:** Single quotes for strings (unless you have a good reason)
3. **Semicolons:** Always. This isn't Python.
4. **Comments:** Sarcastic but informative
5. **Variable names:** camelCase (we're not barbarians)
6. **Function names:** Descriptive but not verbose

**Example:**
```javascript
// Good
function deploySympath(containerId) {
    const violin = getRandomViolin();
    displayViolin(containerId, violin);
}

// Bad
function x(y) {
    var z = a();
    b(y, z);
}
```

### HTML/CSS Style

- Use semantic HTML (because we're not monsters)
- CSS should be inline in the HTML file (yes, we know)
- Comic Sans MS usage is mandatory for violin captions
- Gradients everywhere (this is the way)

### Commit Message Guidelines

We follow a strict commit message format:

```
type: Brief description

Longer description if needed.

Fixes #issue-number
```

**Types:**
- `feat`: New feature (new violin meme)
- `fix`: Bug fix (violin not displaying)
- `docs`: Documentation changes
- `style`: Code style changes (no functional changes)
- `refactor`: Code refactoring
- `test`: Adding tests (just kidding, we don't have tests)
- `chore`: Maintenance tasks

**Examples:**
```
feat: Added violin meme #47

Added a new GIF of a tiny violin being played by a hamster.
This significantly improves our sympathy delivery capabilities.

Fixes #123
```

```
fix: Sad trombone not playing on console.warn

The audio wasn't triggering for console warnings.
Now ALL your problems will be properly acknowledged.

Fixes #456
```

### Pull Request Process

1. **Fork the repo** and create your branch from `main`
2. **Make your changes** following our style guidelines
3. **Test your changes** (open it in a browser, make sure it works)
4. **Update documentation** if you've added features
5. **Write a clear PR description** explaining what you did and why
6. **Wait for review** (we promise to be constructively sarcastic)

**PR Title Format:**
```
[Type] Brief description
```

**Examples:**
- `[Feature] Added 5 new violin memes`
- `[Fix] Corrected audio path bug`
- `[Docs] Updated README with better examples`
