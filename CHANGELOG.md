## Version Naming Convention

We follow semantic versioning: MAJOR.MINOR.PATCH

- **MAJOR:** Incompatible API changes (like making violins bigger)
- **MINOR:** Backwards-compatible new features (like new violin memes)
- **PATCH:** Backwards-compatible bug fixes (like fixing broken violin links)

## Release Notes Format

Each release includes:
- **Added:** New features and capabilities
- **Changed:** Changes to existing functionality
- **Deprecated:** Soon-to-be removed features
- **Removed:** Removed features
- **Fixed:** Bug fixes
- **Security:** Security vulnerability fixes

## Contributing to Changelog

When contributing, please:
1. Add your changes to the [Unreleased] section
2. Follow the format above
3. Use present tense ("Add feature" not "Added feature")
4. Reference issue numbers when applicable
5. Be appropriately sarcastic
